1) Pipeline End-to-End

Flujo principal desde CLI hasta envío NDI con pacing monotónico y canal de retorno opcional.

Entrada CLIVídeo + flags (--dual, metadata RX)
→
Preflightffprobe: resolución, fps, frames
→
Decodeffmpeg: BGRA + audio float32
→
Loop de envíoFrame + bloque audio + overlay opcional
→
Salida NDISeñal principal (+ -Square en dual)
Formato vídeo: BGRA Audio: float32, 48 kHz, 2ch Timing: MonotonicFrameClock

2) Mapa de Módulos

Orquestación

  • stream_video.py coordina setup, loop, envío y cleanup.
  • Gestiona reinicio de decoder si el pipe cae.
  • Activa/desactiva metadata RX por flag.

Decode y metadata multimedia

  • ffmpeg.py: probe_video, start_video_decoder, decode_audio_to_array.
  • Aísla subprocessos de ffmpeg/ffprobe.

Render y sender

  • utils.py: make_sender, draw_square, _pixelate_roi.
  • Overlay solo en modo dual.

Timing

  • core/clock.py evita deriva acumulada.
  • Resincroniza cuando hay overrun.

Backchannel genérico

  • extensions/backchannel/receiver.py captura metadata XML en sender.
  • extensions/backchannel/dispatcher.py enruta mensajes a handlers.

Integración Unity

  • integrations/unity/parsers.py parsea access_transform.
  • integrations/unity/handlers.py loguea eventos parseados.

3) Secuencia de un Frame

1 Leer bytes exactos de vídeo BGRA.
2 Construir np.ndarray writable con .copy().
3 Tomar bloque de audio (con wrap si llega al final).
4 Enviar write_video_and_audio() al sender principal.
5 Si --dual, clonar frame + aplicar draw_square().
6 Drenar metadata RX y despachar handlers.
7 Esperar siguiente deadline con reloj monotónico.

4) Modos de Ejecución

Modo Base

  • 1 salida NDI limpia.
  • Metadata RX activa por defecto.

Modo Dual

  • Salida principal + salida -Square.
  • Más coste por copia/overlay/envío.

Sin metadata RX

  • Flag: --no-rx-metadata.
  • Streamer queda como emisor A/V puro.

Perfil WAN lowres

  • Launcher: Launchers/Stream_NDI_LowRes_WAN.command.
  • Objetivo: más estabilidad en enlaces limitados.
